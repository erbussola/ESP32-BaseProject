# Regole di compilazione del Changelog

Per ogni nuova versione:

- Aggiornare la sezione **Non rilasciato** durante lo sviluppo.
- Prima del rilascio, spostare le modifiche nella nuova versione.
- Inserire la data del rilascio in formato `YYYY-MM-DD`.
- Creare il tag Git corrispondente (es. `v1.1.0`).

Categorie da utilizzare:

- **Aggiunto**
- **Modificato**
- **Corretto**
- **Rimosso**

Non eliminare mai lo storico delle versioni precedenti.
